# Roadmap

## Guiding Principle
Build what proves the loop first. Build what creates lock-in second. Build what drives growth third. Never build V2 features while V1 has unsolved problems.

Discoverability infrastructure (domain, schema markup, App Store listings, content, directory submissions) is part of the Foundation phase — not a post-launch task. The domain needs index history before customers arrive.

### Discoverability
- [ ] Blog publishing cadence active — 2 posts per week minimum
- [ ] Operator profile pages live and indexed as local SEO assets
- [ ] AI Overview appearances for target queries being monitored weekly
- [ ] Keyword rankings for Tier 1 targets tracked weekly

### GTM
- [ ] 15-25 founding operators onboarded in Atlanta (mobile and fixed location)
- [ ] Operators invite existing customers to book through app

### Success Criteria to Move to Public Launch
- 100+ bookings completed
- 4.5+ average rating
- Payment success rate 99%+
- Zero critical bugs in core booking and payment flow
- Operators actively using app to manage their day
- foamauto.com receiving organic traffic (250+ sessions/month)
- App Store listing live and collecting first reviews
